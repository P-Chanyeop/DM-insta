# 센드잇(SendIt) End-to-End 사용자 시나리오 테스트

> 실 사용자가 처음 가입해서 운영까지 가는 흐름을 단계별로 따라가며, 각 단계에서 보이는 화면·동작·예외 상황을 모두 점검한다.

---

## S0. 사전 준비

- 백엔드: `http://localhost:8080` (H2 in-memory, ddl-auto: create-drop)
- 프론트: `http://localhost:5173` (Vite dev)
- 테스트 계정: 매 회 fresh signup (H2가 휘발이라 OK)
- 검증 도구: Claude Preview (스크린샷 + DOM eval)

## S2. 회원가입 + 이메일 인증

**목표**: 신규 사용자 가입 → 이메일 인증 → 온보딩 진입

### 절차
1. `/signup` 접속
2. 이름·이메일·비밀번호 입력 (각 validation 확인)
   - 빈 입력 제출 → 인라인 에러 표시되는가
   - 잘못된 이메일 형식 → 에러 표시
   - 6자 미만 비밀번호 → 에러
3. "무료로 시작하기" 제출
4. 이메일 인증 모달 자동 노출
5. 6자리 코드 입력 (실제 이메일 미발송 환경이면 dev 콘솔/DB 조회)
6. "인증 완료" 클릭 → `/app/onboarding` 자동 이동

### 점검 포인트
- [ ] "로그인 상태 유지" 체크박스 → 실제 동작하는가 (코드상 wired 안 됨 의심)
- [ ] 인증 코드 재발송 작동
- [ ] 비번 찾기 흐름 (`forgotPassword` → `resetPassword`) 종료 후 로그인 화면으로 자동 이동하는가
- [ ] Instagram OAuth 버튼이 `VITE_API_BASE` (잘못된 env 변수명) 사용하는지 확인

## S3. 온보딩 마법사 (5단계)

**목표**: 회원가입 직후 마법사를 끝까지 진행

### Step 0: Instagram 연결
1. "Instagram 계정 연결하기" 클릭 → OAuth URL로 리다이렉트되는가
2. 또는 "건너뛰기" 클릭 → Step 1로 이동
3. URL에 `ig_error=XXX&ig_reason=YYY` 시 에러 박스 표시

### Step 1: 업종 선택
1. 8개 업종 카드 중 하나 선택
2. "다음 단계로" → backend `userService.updateMe({industry})` 호출
3. "건너뛰기" → 업종이 "skipped" 문자열로 저장되는가 (의심: downstream 분기에서 의도와 다를 수 있음)

### Step 2: 키워드 자동 응답
1. 업종 추천 칩 표시되는가
2. 칩 클릭 → 키워드+메시지 자동 채움
3. 직접 키워드+응답 메시지 입력
4. "자동응답 만들기" → `automationService.upsert(DM_KEYWORD)` 호출

### Step 3: 환영 메시지
1. 추천 메시지 카드 클릭 → textarea 자동 채움
2. 환영 메시지 저장 → `automationService.upsert(WELCOME_MESSAGE)`

### Step 4: 성장 도구
1. 3개 체크박스 (commentDM, storyMention, storyReply)
2. 모두 체크 → "활성화하고 완료"
3. FREE 쿼터 5 안에서 정상 생성되는가 (방금 수정한 부분)

### Step 5: 완료 화면
1. 진행 요약 노출
2. "대시보드로 이동" → `markOnboardingComplete()` → `/app`

### 점검 포인트
- [ ] 모든 단계에서 "나중에 할게요" → 확인 없이 즉시 종료 (UX 위험)
- [ ] 성장 도구 생성 실패 시 에러 표시 (현재 silent catch 의심)
- [ ] Step 5에서 "이전" 누르면 Step 4로 돌아가는가
- [ ] 키워드/메시지에 maxlength 없음 (긴 문자열 폭주 가능)
