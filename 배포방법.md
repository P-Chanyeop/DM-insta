# 센드잇 배포 가이드

AWS Lightsail에서 운영 중인 https://sendit.io.kr 재배포 방법.
본인이 자꾸 까먹어서 남기는 메모.

---

## 0. 핵심 정보 (절대 까먹지 말 것)

| 항목 | 값 |
|------|-----|
| **Lightsail 서버 SSH** | `ubuntu@<lightsail-public-ip>` |
| **레포 위치** | `/opt/sendit/repo` (❌ `/home/ubuntu/DM-insta` 아님) |
| **docker-compose 파일** | `docker-compose.prod.yml` (override 아님!) |
| **SSL 터미네이션** | 호스트 시스템 nginx (`/etc/nginx/sites-available/sendit`) |
| **컨테이너 포트** | frontend `127.0.0.1:8081` / backend `127.0.0.1:8080` |
| **DB** | Lightsail Managed MySQL (컨테이너 아님) |

### 경로 헷갈릴 때 빠르게 찾기
```bash
# SSH 접속 후
find / -name "docker-compose.prod.yml" 2>/dev/null
# → /opt/sendit/repo/docker-compose.prod.yml
```

## 1. 자동 배포 (기본)

**main 브랜치에 push하면 GitHub Actions가 자동으로 Lightsail에 배포함.**

로컬에서:
```bash
git add -A
git commit -m "변경 내용"
git push origin main
```

CI 상태 확인:
```bash
gh run list --limit 3
gh run watch    # 현재 진행 중인 워크플로우 실시간 보기
```

보통 **3~5분** 내 배포 완료. 자동 배포가 성공했으면 아래 수동 배포는 필요 없음.

---

## 2. 수동 배포 (CI 실패했거나 긴급 배포)

### 2-1. SSH 접속
```bash
ssh ubuntu@<lightsail-public-ip>
```

### 2-2. 프론트엔드만 변경된 경우 (JSX, CSS, 이미지, MP4 등)
```bash
cd /opt/sendit/repo && \
  git pull origin main && \
  docker compose -f docker-compose.prod.yml up -d --build --no-deps frontend
```
- `--no-deps frontend`: backend 건드리지 않고 frontend만 재빌드 → **서비스 중단 거의 없음**
- 1~2분 소요

### 2-3. 백엔드만 변경된 경우 (Spring Boot 코드, application.yml)
```bash
cd /opt/sendit/repo && \
  git pull origin main && \
  docker compose -f docker-compose.prod.yml up -d --build --no-deps backend
```
- 3~5분 소요 (Gradle 빌드)

### 2-4. 전체 재배포
```bash
cd /opt/sendit/repo && \
  git pull origin main && \
  docker compose -f docker-compose.prod.yml up -d --build
```

### 2-5. 환경변수(.env) 변경한 경우
```bash
cd /opt/sendit/repo && \
  nano .env  # 수정
  # 저장 후:
  docker compose -f docker-compose.prod.yml up -d
```
Docker Compose가 환경변수 변경을 감지해서 자동으로 컨테이너 재생성.

## 3. ⚠️ 절대 하면 안 되는 것 (과거 사고 이력)

### 3-1. `docker compose down` 후 오래 방치 금지
```bash
# ❌ 절대 금지
docker compose down && sleep 60 && docker compose up -d
```
**왜?** `down` 되는 순간 시스템 nginx가 포트 80/443 사용 시도 실패 → HTTPS 먹통. 컨테이너 나가면 `127.0.0.1:8081` 빈 포트 생기는데 시스템 nginx가 proxy_pass 못 해서 502 뜸.

**대신:** `up -d --build` 를 바로 쓰면 Docker가 **재생성 방식**으로 바꾸기 때문에 다운타임 수 초.

### 3-2. `docker-compose.prod.yml` 의 frontend 포트를 `80:80` 으로 바꾸기 금지
```yaml
# ❌ 절대 이렇게 하지 말 것
ports:
  - "80:80"
  - "443:443"
```
**왜?** 호스트 시스템 nginx가 이미 80/443 을 잡고 있어서 docker-proxy가 바인딩 실패 → 컨테이너 크래시 → HTTPS 죽음. (커밋 `584381d` 에서 복구함.)

**항상 유지:**
```yaml
ports:
  - "127.0.0.1:8081:80"
```

### 3-3. `.env` 파일을 git 에 커밋 금지
`.env` 는 `.gitignore` 에 있음. 수정은 SSH 접속 후 서버에서 직접.

## 4. 배포 후 검증

### 4-1. 컨테이너 상태 확인
```bash
docker compose -f docker-compose.prod.yml ps
# frontend 와 backend 가 모두 "Up (healthy)" 이어야 함
```

### 4-2. 로그 확인
```bash
docker compose -f docker-compose.prod.yml logs -f --tail 100 backend
# Ctrl+C 로 나옴
```

### 4-3. HTTP 응답 확인
```bash
# 서버 내부에서
curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8081
# → 200

# 외부에서 (로컬에서)
curl -s -o /dev/null -w "%{http_code}\n" https://sendit.io.kr
# → 200
```

### 4-4. 브라우저 확인
- https://sendit.io.kr 접속해서 랜딩 페이지 정상
- "1분 데모 보기" 클릭 → MP4 재생 (GIF 아님)
- 로그인해서 /app 진입 가능

## 5. 문제 발생 시 트러블슈팅

### 5-1. `bash: cd: /home/ubuntu/DM-insta: No such file or directory`
**원인:** 경로 잘못됨.
**해결:** `/opt/sendit/repo` 사용.

### 5-2. `docker: permission denied`
**원인:** ubuntu 유저가 docker 그룹에 없음.
**해결:** `sudo usermod -aG docker ubuntu && newgrp docker` 후 재로그인.

### 5-3. `https://sendit.io.kr` 이 502 Bad Gateway
**원인:** frontend 컨테이너가 죽음.
**해결:**
```bash
docker compose -f docker-compose.prod.yml logs --tail 50 frontend
# 로그 보고 원인 파악
docker compose -f docker-compose.prod.yml up -d --build --no-deps frontend
```

### 5-4. `git pull` 충돌
**원인:** 서버에서 실수로 파일 수정함.
**해결:**
```bash
# 서버 변경사항 전부 버리고 main 으로 강제 동기화
cd /opt/sendit/repo
git fetch origin
git reset --hard origin/main
docker compose -f docker-compose.prod.yml up -d --build
```

### 5-5. 시스템 nginx 가 죽었을 때
```bash
sudo systemctl status nginx
sudo systemctl restart nginx
sudo nginx -t  # 설정 문법 체크
```
